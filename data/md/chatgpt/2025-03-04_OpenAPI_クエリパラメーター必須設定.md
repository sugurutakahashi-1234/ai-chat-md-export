以下は OpenAPI 3.0 (YAML) 形式の例で、クエリパラメーターを必須に設定しているサンプルです。`status` パラメーターが必須 (required: true) になっています。

```yaml
openapi: 3.0.0
info:
  title: Example API
  version: "1.0.0"
paths:
  /users:
    get:
      summary: Retrieve a list of users
      parameters:
        - in: query
          name: status
          required: true         # クエリパラメーターを必須に設定
          description: The user status to filter
          schema:
            type: string
            enum:
              - active
              - inactive
      responses:
        '200':
          description: OK
          content:
            application/json:
              schema:
                type: array
                items:
                  $ref: '#/components/schemas/User'
        '400':
          description: Bad Request
components:
  schemas:
    User:
      type: object
      properties:
        id:
          type: integer
          example: 1
        name:
          type: string
          example: John Doe
        status:
          type: string
          example: active
```

上記の例では、`/users` に対する `GET` のクエリパラメーター `status` が必須項目 (`required: true`) となっています。このように記述することで、クライアントがリクエストを送る際に、`status` クエリパラメーターが指定されていない場合はバリデーションエラーを返すような実装を行うことができます。
